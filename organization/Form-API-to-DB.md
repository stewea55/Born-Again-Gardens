## Sponsors → `sponsors_public` (display only; single table)

**Date:** 2026-03-01

One table for "Our Generous Sponsors" on the home page. Not used by checkout. Updated via admin dashboard or manually. Payment is handled offline (contact info@bornagaingardens.org, 317-385-4165). **RLS:** All columns in this table (including company_name, tier, company_url, logo) use the same policies: SELECT for `anon` and `authenticated` (so the page can show logos and tier); INSERT/UPDATE/DELETE for `authenticated` where `is_admin()` (same pattern as other admin tables).

| Form field / API payload | DB column   | Notes |
|---------------------------|------------|-------|
| company_name              | company_name |       |
| tier                      | tier       | optional |
| company_url               | company_url| optional |
| logo                      | logo       | optional; image URL for sponsor logo |
| display_order             | display_order | optional; stacking order on the sponsors canvas when items overlap |
| layout                    | layout     | optional jsonb; nested canvas objects: `logo` and `name` with independent position + size (`centerX`, `centerY`, `width`, `height`); name includes `fontSize` |
| (id)                      | id         | uuid; generated by admin API on insert if omitted |
| (created_at)              | created_at | default now() |

**Date:** 2026-03-02

Admin-only sponsors canvas editor writes placement/sizing to `layout` and overlap order to `display_order`.
Clicking X in the editor clears only the selected object (`layout.logo` or `layout.name`). If both are null, sponsor is hidden from home, but the row is still kept.

## Authenticated cart APIs → `profiles.cart` and `profiles.basket`

For signed-in users, cart state is persisted in **profiles** (jsonb). Only `cart` and `basket` columns are updated by the cart API; `role` is never writable by the app.

| API payload | DB column | Notes |
|-------------|-----------|-------|
| cart_type (from request) | — | `shop` → update `profiles.cart`, `harvest` → update `profiles.basket` |
| items (array) | cart or basket | jsonb array of line items (item_type, item_id, quantity, size, unit_price, metadata) |
| user (from auth token) | id | row is `profiles.id = auth.uid()` |

## Other tables (reference)

- **transaction:** Links to user_id (profiles), guest_id (guests), or sponsor_id (sponsors_public). Columns: payment, donation_amount, stripe_id, status, etc. For logged-in checkout, user_id is set and guest_id is null.
- **harvest_quantity:** Written by server on successful payment (webhook): for each basket line item, upsert by plant_id (add quantity; set plant_name). Columns: id, plant_id, plant_name, quantity, created_at.
- **plant_catalog:** Read-only for catalog; columns include name, scientific_name, category, harvest_start, harvest_end, market_price, etc.
- **resources:** id, resource_name, image_url, page, resource_type, created_at (for images/files used on the site).
- **hero_images:** id, page, image_url, created_at.
- **profiles / user_preference:** Auth/user profile data; see Supabase schema for columns.
